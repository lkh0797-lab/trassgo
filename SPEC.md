## 1. 프로젝트 목적

한국 관세청 시군구별 품목별 수출 통관 데이터를 종목 단위로 매핑하여,
**증권사 리포트보다 1~2개월 먼저 수출 모멘텀 변화를 포착**하는 개인용 트래킹 시스템 구축.

핵심 가치:
- **남들이 잘 안 보는 데이터**: HS 6단위 + 시군구 분해
- **빠른 신선도**: 5월 1일에 4월 잠정치, 5월 15일에 정밀치
- **종목 단위 자동 추정**: 청주(에코프로비엠) / 포항(포스코퓨처엠+에코프로비엠) 같은 시군구 합산
- **확장성**: 종목 추가는 사용자/클로드 대화로 5~20분 매핑 → 그 다음 자동화

## 3. 데이터 소스 (확정)

### 3.1 관세청 OpenAPI (data.go.kr) — ✅ 키 발급 완료

| API | 엔드포인트 | 용도 |
|---|---|---|
| **시군구별 품목별 수출입실적** | `/1220000/sigunguperprlstperacrs/getSigunguPerPrlstPerAcrs` | 종목별 추정 (메인) |
| **품목별 국가별 수출입실적** | `/1220000/nitemtrade/getNitemtradeList` | 수출국가 분해 (보조) |

- **호출 제한**: API당 일 10,000건 (충분)
- **요금**: 무료
- **응답 포맷**: XML
- **시계열 한도**: 1회 호출당 최대 12개월

### 3.2 종목 유니버스 마스터
- **소스**: 사용자 제공 엑셀 (`개별종목_시총1500억이상_TOP500_*.xlsx`)
- **갱신 주기**: 사용자 수동 (월 1회 또는 필요 시)
- **편입/편출**: 자동 처리하지 않음, 사용자 직접 결정

### 3.3 종목 ↔ HS코드 ↔ 시군구 매핑
- **저장**: `data/master/stock_mapping.csv`
- **추가 워크플로우**: 사용자 ↔ 클로드 대화로 매핑 → 사용자 승인 → 추가
- **LLM 매핑 자동화**: ❌ 사용 안 함 (사용자 결정)

### 3.4 DART OpenAPI — Phase 2에서 추가
- 종목별 매출비중 자동 추출용 (다종목 매핑 시 가중치)
- 키 미발급, 필요 시점에 발급 추진

## 5. 종목 매핑 마스터 (`stock_mapping.csv`)

| 컬럼 | 예시 | 설명 |

| stock_code | 247540 | 종목코드 |

| hs_code | 284190 | HS 6단위 |
| hs_desc | 산의 금속산염류 (양극재) | HS 한글 |

| sgg_keyword | 청주시 | 시군구 필터 키워드 |

## 6. 엑셀 대시보드 구조 (확정)

파일: `수출입통계_트래커_2026-05.xlsx`

### 시트 목록
| 시트 | 내용 |
|---|---|
| **Summary** | 매핑 종목 전체 — 종목별 1행, 최근월 수출액·YoY·시그널 표시 |
| **Universe** | 500종목 + 매핑상태 컬럼 (사용자 입력 엑셀 그대로) |
| **HS_Mapping** | 매핑 마스터 (stock_mapping.csv 그대로) |
| **{종목명}_Detail** | 매핑된 종목 1개당 시트 1개 — Dashboard 형식 (KPI + 시계열 + 차트) |
| **Raw_Data** | 관세청 원본 (충북+경북+...모든 매핑된 시도) |

종목 시트는 매핑된 만큼만 생성 (예: 5종목 매핑 → 5개 종목 시트).

## 7. 시그널 정의 (Phase 1에서는 단순화)

v0.2 SPEC의 5개 시그널 → v0.3에서는 **3개로 축소** (단순성 우선):

| 시그널 | 정의 | 텔레그램 푸시 |
|---|---|---|
| **MOMENTUM** | YoY ≥ +30% AND 3개월 연속 양수 | ⭐ 즉시 |
| **BREAKOUT** | 24개월 최고치 갱신 | ⭐ 즉시 |
| **WARN** | YoY ≤ -25% AND 3개월 연속 감소 | 보유 종목만 |

ACCELERATION/REVERSAL은 데이터 누적 후 v0.4에서 추가 검토.

## 9. 종목 추가 워크플로우 (대화형)

사용자가 클로드에게 종목 추가 요청 시:

```
사용자: "한미반도체 추가해줘"
클로드:
  1. 사업보고서/공시 확인 → 주력제품 추정
  2. HS코드 후보 제시 (예: 8486.20 반도체장비)
  3. 생산거점 시군구 확인 (예: 인천)
  4. 검증: 해당 시도의 HS코드 데이터 fetch → 회사 매출 규모와 대조
  5. 사용자에게 매핑 제안 + 검증 결과 공유
사용자: "OK"
클로드:
  6. stock_mapping.csv에 추가
  7. 36개월 데이터 fetch (캐시)
  8. 다음 정기 실행에 자동 포함됨
```

→ 종목 추가는 *시스템 빌드 후* 1종목당 **5~20분** 소요 (검증 포함)

## 11. 단위·표기 규칙 (확정)

- **금액 단위**: 모든 화폐 수치는 **천 USD ($1,000)** 로 저장 → 엑셀 표시 시 헤더에 `(천USD)` 또는 변환 후 `$XX.XM` 표기
- **퍼센트**: YoY/MoM은 소수점 1자리, +/- 기호 표시 (`+12.3%`, `-5.4%`)
- **연월**: `YYYY.MM` 통일 (관세청 API 응답과 동일)
- **종목코드**: 6자리 문자열 (앞 0 보존)
- **HS코드**: 6단위는 6자리 문자열 (`284190`), 10단위는 10자리
